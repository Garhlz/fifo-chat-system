# AGENTS

本仓库的协作与提交约定如下。

## 1. 文档同步要求

每次修改代码、脚本、目录结构、运行方式或功能行为后，必须同步检查并更新相关文档，至少包括：

- `README.md`
- `docs/requirement.md`
- `docs/TODO.md`

执行原则：

- 如果实现范围、使用方式、依赖、输出结果或项目结构发生变化，必须同步更新文档。
- 如果某项 TODO 已完成，需及时勾选或调整对应条目。
- 如果本次改动不影响现有文档，也要在提交前明确检查一次文档是否仍然准确。

## 2. 提交信息格式

提交信息必须使用规范格式：

```text
<type>(<scope>): <summary>
```

示例：

```text
feat(user): 添加第三方 OAuth 登录接口
```

常用 `type` 包括：

- `feat`
- `fix`
- `docs`
- `refactor`
- `test`
- `chore`

`summary` 要求：

- 使用简洁明确的中文描述
- 直接说明本次改动的核心内容
- 避免空泛表述，如“更新一下”“修复问题”

## 3. 提交说明内容

除提交标题外，建议补充提交说明，概述本次改动的关键点。推荐格式如下：

```text
feat(user): 添加第三方 OAuth 登录接口

- 接入 GitHub 和 Google 登录 API
- 优化了登录页面的 UI 布局
- 添加了相关的用户信息缓存逻辑
```

要求：

- 每条说明聚焦一个明确改动点
- 优先描述功能、行为、接口、数据结构或文档变化
- 保持与实际提交内容一致，不写未完成事项

## 4. 执行要求

- 在准备提交前，先检查代码改动与文档是否一致。
- 在准备提交前，确认提交标题和提交说明符合上述格式。
- 如改动较大，应保证提交说明覆盖主要影响点。
- 提交信息中不要添加 Co-Authored-By 或其他作者标签。
